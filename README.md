# Meteo Project

Приложение позволяет пользователям выполнять поиск погоды по названию города, поддерживается автодополнение.
Хранит историю поиска как для авторизованных пользователей, так и для неавторизованных пользователей.

---

## Функционал

- Поиск погоды по городу
- Автозаполнение названий городов
- Прогноз погоды на выбранное количество дней
- История поиска для авторизованных пользователей
- История поиска для анонимных пользователей
- Простой JSON API для статистики поиска по городам
- Написаны тесты

---

## Стэк

- Python 3.12
- Django 5.2
- SQLite
- Docker
- Docker Compose

---

## Запуск веб-приложения

Ниже приведены два способа локального запуска проекта.

### Вариант 1: Загрузите проект в виде архива

1. Нажмите **Code → Download ZIP**.
2. Извлеките архив в любое место .
3. Откройте терминал в корне проекта.

### Вариант 2: Клонируйте репозиторий, используя git

1. ``git clone https://github.com/gogotye/meteo_project.git``
2. ```cd meteo_project```.

---

### Создайте и активируйте виртуальную среду:

```python -m venv venv```

#### Для Windows
- venv\Scripts\activate

#### Для Linux/Mac
- source venv/bin/activate

---

### Установите зависимости из файла requirements.txt
```pip install -r requirements.txt```

---

### Далее, нужно принять миграции и запустить сервер
1. ```python manage.py migrate```
2. ```python manage.py runserver```

### Готово! Откройте в браузере http://127.0.0.1:8000

---

## Быстрый запуск с помощью Docker

Самый простой способ запустить приложение - использовать Docker.


- Нужно установить [Docker Desktop](https://www.docker.com/get-started/) и открыть его
- Открыть терминал
- Использовать команду ```docker run -p 8000:8000 -v django_db:/django-app/db gogotye/meteo_project```
- Открыть в браузере http://localhost:8000/search/

### Альтернатива: запуск через Docker Compose

Если вы клонировали репозиторий проекта, то можно использовать Docker Compose:

- Открыть терминал в корне проекта
- `docker compose up`
